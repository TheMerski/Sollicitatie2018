Inleiding 
We hebben twee soorten entiteiten: 
- Klanten  
- Producten  

Een klant heeft een klantnummer en naam en kan optioneel onder een andere klant vallen (sub-vestigingen). Een klant kan optioneel producten hebben. Een product heeft een productnummer en een naam. 

Opdracht 
Maak een overzicht op één pagina dat deze data-opzet weerspiegelt. Zorg daarnaast dat de naam van een product naar een achterliggende detailpagina verwijst waarop enkel de naam van het product wordt getoond.   
Bied onder het overzicht enerzijds de mogelijk om een nieuw klant toe te voegen, anderzijds om een nieuw product toe te voegen. Hou hierbij rekening met de hiërarchische positie waarbinnen het nieuwe object toegevoegd moet worden. 
De opmaak moet enigszins toonbaar zijn en bruikbaar genoeg om in toekomst verder te kunnen stijlen (bijvoorbeeld door een designer). Verder zien we graag terug dat je rekening houdt met een stukje veiligheid tegen malafide code.  

Zeker niet onbelangrijk: hou je code simpel en overzichtelijk. Onderbouw waar mogelijk je eigen toevoegingen op deze opdracht  in een README
